Software Requirements Specification (SRS)

Project Title: Deepfake Detection System using MesoNet
Version: 1.0
Date: April 12, 2025
Prepared by: The DeepFake Detection Team

1. Introduction

1.1 Purpose

This document specifies the requirements for the Deepfake Detection System, which aims to identify manipulated images using MesoNet CNN architecture.

1.2 Scope

The system processes uploaded images, analyzes them using a pre-trained MesoNet model, and reports whether the image contains a real face or a deepfake.

1.3 Definitions and Acronyms

  • Deepfake: AI-generated synthetic media
  • MesoNet: CNN architecture for deepfake detection
  • CNN: Convolutional Neural Network
  • UI: User Interface
  • API: Application Programming Interface

1.4 References

  • MesoNet: Afchar et al., 2018
  • Flask Documentation
  • OpenCV Documentation

2. Overall Description

2.1 Product Perspective

A web-based application with an ML back end and user-friendly front end for deepfake classification.

2.2 Product Features

  • Image upload interface
  • Face detection preprocessing
  • Classification with confidence scores
  • Detection history and educational tools

2.3 User Classes

  • General Users
  • Students/Researchers
  • Educators

2.4 Operating Environment

Chrome, Firefox, Safari, Python 3.11+, Flask, Gunicorn, PostgreSQL, TensorFlow/Keras

2.5 Design Constraints

  • Use MesoNet model
  • Max image processing time: 5 sec
  • Responsive web interface

2.6 Assumptions

  • Pre-trained MesoNet available
  • Internet connectivity assumed

3. Specific Requirements

3.1 Functional Requirements

  • Image validation (FR1.2), face detection (FR2.1), classification (FR3.1), education (FR4.1)

3.2 Non-Functional Requirements

  • Performance: <5 sec processing, 85% accuracy
  • Security: HTTPS, no permanent storage
  • Usability: Responsive, no prior training
  • Reliability: Graceful error handling, 99% uptime

4. External Interface Requirements

  • UI Pages: Homepage, Results, About, Quiz, Gallery
  • Software Interfaces: PostgreSQL, OpenCV, TensorFlow
  • Communications: HTTP/HTTPS
